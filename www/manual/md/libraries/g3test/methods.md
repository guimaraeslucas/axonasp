# G3TESTSUITE Methods

## Overview
This page provides a summary of the methods available in the **G3TESTSUITE** library for automated testing in G3Pix AxonASP. All assertion methods evaluate their conditions using native VBScript coercion and standard AxonASP Virtual Machine semantics.

## Method List

- **AssertEmpty**: Checks if a value is `Empty`. Returns a **Boolean**.
- **AssertEqual**: Checks if two values are equal. Returns a **Boolean**.
- **AssertFalse**: Checks if a condition is `False`. Returns a **Boolean**.
- **AssertLength**: Checks the length or count of a value. Returns a **Boolean**.
- **AssertNothing**: Checks if a value is an uninitialized object. Returns a **Boolean**.
- **AssertNotEqual**: Checks if two values are not equal. Returns a **Boolean**.
- **AssertNull**: Checks if a value is `Null`. Returns a **Boolean**.
- **AssertRaises**: Checks if a code block raises an error. Returns a **Boolean**.
- **AssertTrue**: Checks if a condition is `True`. Returns a **Boolean**.
- **AssertTypeName**: Checks the `TypeName` of a value. Returns a **Boolean**.
- **Describe**: Sets the label for a group of tests. Returns **Empty**.
- **Fail**: Explicitly marks an assertion as failed. Returns **Boolean** (False).

## Signatures and Parameters

### AssertEmpty(value [, message])
- **value** (Variant): The variable to test for `Empty` status.
- **message** (String): An optional diagnostic message to display on failure.

### AssertEqual(expected, actual [, message])
- **expected** (Variant): The baseline value for comparison.
- **actual** (Variant): The value produced by the system under test.
- **message** (String): An optional diagnostic message to display on failure.

### AssertRaises(code [, expected] [, message])
- **code** (String): The VBScript statement to execute.
- **expected** (Integer/String): Optional expected error number or partial error message.
- **message** (String): An optional diagnostic message to display on failure.

### Describe(label)
- **label** (String): A descriptive name for the current test block.

## Remarks
- Use the **Describe** method to categorize assertions. This simplifies the analysis of structured test reports generated by the `axonasp-testsuite` runner.
- The **AssertEqual** method is binary-safe and can compare strings, numbers, dates, and arrays.
- All assertion methods increment the internal **Total** counter. Failed assertions increment the **Failed** counter, while successful ones increment the **Passed** counter.
