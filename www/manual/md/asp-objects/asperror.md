# ASPError Object

## Overview

The `ASPError` object provides structured information about an error that occurred in an ASP script. You obtain a reference to it by calling `Server.GetLastError()`. This object is most commonly used inside a custom error handler page (`500-100.asp` or equivalent) assigned in `web.config`.

All properties on `ASPError` are **read-only** â€” they describe the error that was captured and cannot be modified.

---

## Properties

### ASPCode

Returns the error code generated by AxonASP's scripting engine. This is distinct from `Number`, which holds the VBScript/Win32 HRESULT error number.

**Type:** Integer (read-only)

**Example:**
```asp
<%
Dim e : Set e = Server.GetLastError()
Response.Write "ASP Code: " & e.ASPCode & "<br>"
%>
```

---

### Number

Returns the numeric error code, typically matching the VBScript or Win32 HRESULT value. For example, VBScript runtime error 11 (Division by zero) returns `11`; ActiveX errors include a facility code.

**Type:** Integer (read-only)

**Example:**
```asp
<%
Dim e : Set e = Server.GetLastError()
Response.Write "Error Number: " & e.Number
%>
```

---

### Category

Returns a string that classifies the error origin. Common values include:

| Value | Meaning |
|-------|---------|
| `"ASP"` | Error from the ASP scripting engine itself |
| `"VBScript runtime"` | Error thrown at VBScript runtime (e.g., type mismatch) |
| `"VBScript compilation"` | Syntax error caught at compile time |
| `"AxonASP"` | Internal AxonASP server error |

**Type:** String (read-only)

**Example:**
```asp
<%
Dim e : Set e = Server.GetLastError()
Response.Write "Category: " & e.Category
%>
```

---

### Description

Returns the human-readable description of the error, equivalent to `Err.Description` in VBScript but captured by the error handler.

**Type:** String (read-only)

**Example:**
```asp
<%
Dim e : Set e = Server.GetLastError()
If e.Number <> 0 Then
    Response.Write "Description: " & Server.HTMLEncode(e.Description)
End If
%>
```

---

### ASPDescription

Returns an extended description specific to the ASP layer, which may include additional context beyond the VBScript error message. Often the same as `Description`; occasionally more detailed.

**Type:** String (read-only)

---

### Source

Returns the source code fragment or identifier that triggered the error, as reported by the scripting engine. For VBScript errors, this is typically the VBScript engine name.

**Type:** String (read-only)

---

### File

Returns the virtual path of the ASP file in which the error occurred. Useful for logging and display in error pages.

**Type:** String (read-only)

**Example:**
```asp
<%
Dim e : Set e = Server.GetLastError()
Response.Write "Error in: " & e.File
%>
```

---

### Line

Returns the line number within the ASP file where the error occurred. Combined with `Column`, this locates the exact position of the fault.

**Type:** Integer (read-only)

**Example:**
```asp
<%
Dim e : Set e = Server.GetLastError()
Response.Write "Line: " & e.Line & ", Column: " & e.Column
%>
```

---

### Column

Returns the column number on the error line. May be `0` if the scripting engine does not provide column-level precision.

**Type:** Integer (read-only)

---

### HelpFile

Returns the path to a help file associated with the error, if one exists. Typically empty in AxonASP.

**Type:** String (read-only)

---

### HelpContext

Returns the help context identifier within the help file. Typically `0` in AxonASP.

**Type:** Integer (read-only)

---

## Usage Pattern â€” Custom Error Page

The most common use of `ASPError` is in a dedicated error page. You configure this page via `Response.Status` or through `web.config` error page mappings, and it calls `Server.GetLastError()` to display meaningful information.

**Example â€” error.asp:**
```asp
<%
Dim e
Set e = Server.GetLastError()

Response.Status = "500 Internal Server Error"
%>
<!DOCTYPE html>
<html>
<head><title>Server Error</title></head>
<body>
<h1>An error occurred</h1>
<table border="1">
<tr><th>Property</th><th>Value</th></tr>
<tr><td>Category</td><td><%= Server.HTMLEncode(e.Category) %></td></tr>
<tr><td>Error Number</td><td><%= e.Number %></td></tr>
<tr><td>Description</td><td><%= Server.HTMLEncode(e.Description) %></td></tr>
<tr><td>File</td><td><%= Server.HTMLEncode(e.File) %></td></tr>
<tr><td>Line</td><td><%= e.Line %></td></tr>
<tr><td>Column</td><td><%= e.Column %></td></tr>
</table>
</body>
</html>
```

---

## Remarks

- `Server.GetLastError()` always returns an `ASPError` object. When no error has occurred, all numeric properties are `0` and string properties are empty.
- In production, avoid displaying raw error details to end users; instead log them using `Response.AppendToLog` and show a user-friendly message.
- When `enable_asp_debugging` is set to `false` in `axonasp.toml`, AxonASP suppresses detailed error output in HTTP/FastCGI modes; use `Server.GetLastError()` in your custom error handler to still capture the data.
