Prerequisites

Before installing G3Pix AxonASP, ensure you have the following:

Required

  • Go 1.21 or higher - Download from golang.org
  • Git - For cloning the repository

Recommended

  • Basic understanding of Classic ASP and VBScript
  • Familiarity with command-line interfaces
  • Text editor or IDE (VS Code, Sublime Text, etc.)
üí° Note: G3Pix AxonASP is a standalone application with no external dependencies beyond Go itself.

Installation

Step 1: Clone the Repository

git clone https://github.com/guimaraeslucas/axonasp.git
cd axonasp

Step 2: Download Dependencies

go mod download

Step 3: Run the Development Server

go run main.go
‚úÖ Success! Your server should now be running at http://localhost:4050

Configuration

G3Pix AxonASP uses environment variables for configuration. Create a .env file in the root directory:

# Server Configuration
SERVER_PORT=4050
WEB_ROOT=./www
TIMEZONE=America/Sao_Paulo
DEFAULT_PAGE=default.asp
SCRIPT_TIMEOUT=30
DEBUG_ASP=FALSE

# SMTP Configuration (Optional)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password

Configuration Options

Variable Default Description
SERVER_PORT 4050 HTTP server port
WEB_ROOT ./www Root directory for ASP files
TIMEZONE America/Sao_Paulo Server timezone
DEFAULT_PAGE default.asp Default document name
SCRIPT_TIMEOUT 30 Script execution timeout (seconds)
DEBUG_ASP FALSE Enable HTML stack traces
SMTP_HOST - SMTP server hostname
SMTP_PORT 587 SMTP server port
SMTP_USER - SMTP authentication username
SMTP_PASS - SMTP authentication password

Building for Production

Build for Current Platform

go build -o axonasp

This creates an executable for your current operating system.

Windows (64-bit)

GOOS=windows GOARCH=amd64 go build -o axonasp.exe

Run the executable:

axonasp.exe

Linux (64-bit)

GOOS=linux GOARCH=amd64 go build -o axonasp

Make it executable and run:

chmod +x axonasp
./axonasp

macOS (Intel)

GOOS=darwin GOARCH=amd64 go build -o axonasp

macOS (Apple Silicon)

GOOS=darwin GOARCH=arm64 go build -o axonasp

Make it executable and run:

chmod +x axonasp
./axonasp

Reverse Proxy Setup

For production deployments, use a reverse proxy like Nginx, Apache, or IIS in front of AxonASP.

Nginx Configuration

server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:4050;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Apache Configuration

<VirtualHost *:80>
    ServerName yourdomain.com
    
    ProxyPreserveHost On
    ProxyPass / http://localhost:4050/
    ProxyPassReverse / http://localhost:4050/
    
    <Proxy *>
        Order deny,allow
        Allow from all
    </Proxy>
</VirtualHost>

IIS URL Rewrite

Install ARR (Application Request Routing) and URL Rewrite modules, then add to web.config:

<configuration>
    <system.webServer>
        <rewrite>
            <rules>
                <rule name="ReverseProxy" stopProcessing="true">
                    <match url="(.*)" />
                    <action type="Rewrite" url="http://localhost:4050/{R:1}" />
                </rule>
            </rules>
        </rewrite>
    </system.webServer>
</configuration>

Running Multiple Instances

G3Pix AxonASP supports one application per instance due to global.asa loading. To host multiple applications, run multiple instances on different ports.

# Terminal 1 - Application 1
SERVER_PORT=4050 WEB_ROOT=./app1/www ./axonasp

# Terminal 2 - Application 2
SERVER_PORT=4051 WEB_ROOT=./app2/www ./axonasp

# Terminal 3 - Application 3
SERVER_PORT=4052 WEB_ROOT=./app3/www ./axonasp

Then configure your reverse proxy to route different domains to different ports.

Troubleshooting

Port Already in Use

If you see "address already in use", change the SERVER_PORT in your .env file.

ASP Errors Not Showing

Set DEBUG_ASP=TRUE in your .env file to see detailed error messages.

Database Connection Issues

Ensure your database drivers are properly configured and connection strings are correct.

Session Not Persisting

Check that the temp/session directory exists and is writable.
